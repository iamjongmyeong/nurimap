# Browse And Detail

## Scope
- 지도 탐색
- 목록 탐색
- 목록 영역 전환
- 데스크톱 상세 패널
- 모바일 상세 화면
- 상세 진입과 복귀

## Browse Flow
1. 사용자가 지도 화면을 본다.
2. 데스크톱에서는 목록에서 장소를 선택하거나 지도 마커를 클릭한다.
3. 모바일에서는 `목록 보기` 버튼으로 장소 목록 페이지로 이동하거나 지도 마커를 클릭한다.
4. 목록에서는 평균 별점, 리뷰 수, 제로페이 가능 여부를 함께 확인한다.
5. 데스크톱에서는 목록 영역 안의 내용이 상세 화면으로 전환된다.
6. 모바일에서는 전체 화면 상세 페이지가 열린다.
7. 사용자가 `장소 추가`를 누르면 기존 목록 영역이 등록 화면으로 바뀐다.
8. 등록 화면을 닫으면 사용자는 직전의 목록 탐색 맥락으로 돌아간다.
9. 상세에서 평균 별점, 별점 수, 장소 구분, 제로페이 가능 여부, 리뷰를 본다.
10. 데스크톱에서는 닫기 액션으로, 모바일에서는 뒤로 가기 액션으로 지도 화면으로 돌아간다.

## Rules
- 데스크톱 상세는 별도 floating panel이 아니라 기존 목록 영역 안에서 열린다.
- 모바일 상세는 전체 화면 페이지로 열어야 한다.
- 모바일의 화면 뒤로 가기 버튼과 브라우저 기본 뒤로 가기는 같은 동작을 해야 한다.
- 모바일에서 지도 화면으로 돌아갈 때 선택한 장소와 지도 위치 맥락을 유지해야 한다.
- 목록과 지도는 같은 장소 선택 상태를 공유해야 한다.
- 장소 등록은 목록 영역의 내용 전환으로 처리하고, 별도 전용 등록 화면을 추가하지 않는다.
- 이번 1차 browse/detail UI에서는 네이버 지도 이동, 추천, 내 리뷰, 리뷰 작성 UI를 상세에 노출하지 않는다.

## Failure Expectations
- 목록, 상세, 지도 로딩 실패는 현재 맥락을 이해할 수 있는 방식으로 드러나야 한다.
- 상세 진입 실패가 나도 사용자가 탐색 상태를 잃지 않아야 한다.
- 등록 화면에서 주소 geocoding이나 저장이 실패해도 사용자가 목록 영역 맥락을 잃지 않아야 한다.
