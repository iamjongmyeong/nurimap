# UX/UI Design

## Global Layout Rules
- breakpoint: `768px`
- 지도 시작 위치
  - 주소: `서울 마포구 양화로19길 22-16`
  - 좌표: `37.558721, 126.924440`

## Authentication Flow
1. 사용자가 웹사이트에 접속한다.
2. 로그인 화면에서 회사 이메일을 입력한다.
3. 시스템이 허용 도메인인지 확인한다.
4. 이메일로 로그인 전용 URL을 발송한다.
5. 사용자가 로그인 전용 URL을 클릭한다.
6. 로그인 전용 URL이 유효하면 place 탐색 화면으로 이동한다.
7. 같은 브라우저 재방문 시 저장 세션이 유효하면 로그인 화면을 건너뛴다.

로그인 링크 규칙:
- 로그인 전용 URL은 직접 로그인에 사용할 수 있다.
- 로그인 전용 URL은 5분 동안만 유효하다.
- 새 로그인 링크를 발급하면 이전 로그인 전용 URL은 무효화된다.
- 로그인 전용 URL은 한 번 사용하면 다시 사용할 수 없다.

## Place Submission Flow
1. 로그인 사용자가 place 추가 UI를 연다.
2. 네이버 지도 URL을 입력한다.
3. 시스템이 URL을 정규화하고 place 데이터를 추출한다.
4. 시스템은 좌표 확보를 아래 순서로 시도한다.
   - 네이버 응답의 위도/경도
   - `road_address` geocoding
   - `land_lot_address` geocoding
5. 네이버 응답 실패 또는 좌표 확보 실패 시 alert를 표시하고 등록 프로세스를 종료한다.
6. 좌표 확보에 성공하면 사용자가 `place_type`, `zeropay_status`, 초기 별점, 리뷰를 입력한다.
   - 초기 별점 입력 UI는 별 모양 버튼을 사용한다.
   - 초기 상태는 5점이 기본 선택되어 있다.
7. 시스템이 중복 place를 확인한다.
8. 신규 생성 또는 기존 place에 정보 누적 후 완료 상태를 보여준다.
9. 지도와 목록이 최신 상태로 갱신된다.

## Browse Flow
1. 사용자가 지도와 목록을 본다.
2. 목록에서 place를 선택하거나 지도 마커를 클릭한다.
3. 목록에서는 평균 별점과 리뷰 수를 함께 확인할 수 있다.
4. 상세 화면이 열린다.
5. 상세에서 평균 별점, 별점 수, 등록자, 추천 수, 리뷰, 네이버 지도 이동 버튼을 본다.
6. 닫기 또는 뒤로 가기로 탐색 상태로 돌아간다.

## Rating Flow
1. 로그인 사용자가 상세 화면의 별 모양 버튼 중 하나를 선택한다.
2. 시스템이 1점에서 5점 사이의 정수인지 확인한다.
3. 기존 별점이 없으면 새 별점을 생성한다.
4. 기존 별점이 있으면 해당 사용자의 별점을 수정한다.
5. 평균 별점과 별점 수가 갱신된다.

## Recommendation Flow
1. 로그인 사용자가 상세 화면의 추천 버튼을 누른다.
2. 기존 추천이 없으면 추천을 추가한다.
3. 기존 추천이 있으면 추천을 취소한다.
4. 추천 수가 갱신된다.

## Review Flow
1. 로그인 사용자가 상세 화면에서 리뷰를 입력한다.
2. 시스템이 저장 후 최신 리뷰 목록을 다시 보여준다.
3. 작성자와 작성일이 함께 표시된다.

## Desktop Layout `>= 768px`

### Base State
- 지도를 전체 화면으로 표시한다.
- 왼쪽에 목록 사이드바를 띄운다.
  - 너비: `390px`
  - 높이: 전체 높이
- 사이드바에는 접기/펼치기 버튼이 있다.
- place 추가 버튼은 사이드바 상단에 배치한다.
  - 크기: `342px x 48px`
  - floating button은 사용하지 않는다.

### Detail State
- 목록 또는 지도에서 place 선택 시 상세 패널을 연다.
- 상세 패널은 사이드바 오른쪽에 위치한다.
  - 너비: `390px`
  - 높이: `calc(100vh - 48px)`
  - 사이드바와 간격: `24px`
- X 버튼으로 닫는다.

## Mobile Layout `< 768px`

### Base State
- 지도를 전체 화면으로 표시한다.
- 목록은 하단 바텀시트 형태로 표시한다.
  - 초기 높이: 화면 높이의 약 `20%`
  - 위로 스와이프 시 전체 화면 가까이 확장
  - 목록 내부는 스크롤 가능

### Detail State
- 목록 또는 지도에서 place 선택 시 상세 화면을 연다.
- X 버튼으로 닫는다.
- 상세를 닫으면 이전 목록 탐색 상태로 돌아간다.

## UI State Model
| State | Description |
|---|---|
| `auth_required` | 로그인 전 상태 |
| `auth_link_sent` | 로그인 링크 발송 후 대기 상태 |
| `map_browse` | 지도/목록 탐색 기본 상태 |
| `place_add_open` | place 등록 UI가 열린 상태 |
| `place_detail_open` | place 상세가 열린 상태 |
| `mobile_sheet_peek` | 모바일 목록 바텀시트 기본 상태 |
| `mobile_sheet_expanded` | 모바일 목록 바텀시트 확장 상태 |

## Interaction Notes
- 지도 라벨은 기본적으로 `level 1-5`에서 표시하고 `level 6`부터 숨긴다.
- place 상세에서 Naver 이동 버튼은 항상 일관된 위치에 둔다.
- 별점 입력 UI는 숫자 입력 필드가 아니라 별 모양 버튼을 사용한다.
- place 추가 화면에서는 5번째 별이 기본 선택 상태여야 한다.
- 모바일에서 상세 화면과 목록 시트의 전환은 사용자가 맥락을 잃지 않도록 유지해야 한다.
