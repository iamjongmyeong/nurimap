# Spec: Place Detail

## Summary
place 상세 화면에 표시해야 할 정보와 상세 화면 동작을 정의한다.

## Scope
- 상세 표시 필드
- 데스크톱 상세 닫기 동작
- 데스크톱 목록 영역 전환형 상세
- 모바일 상세 표현 방식
- 모바일 뒤로 가기 동작
- 상세 로딩/오류 상태

## Functional Requirements
- 상세 화면에는 이름, 주소, 종류, 평균 별점, 별점 수, 제로페이 가능 여부, 리뷰를 표시한다.
- 화면에 표시하는 `별점 수`는 내부 집계 필드 `review_count`를 의미한다.
- 데스크톱과 모바일에서 동일한 정보 구조를 유지한다.
- 데스크톱 상세는 별도 floating panel이 아니라 기존 목록 영역 안에서 열린다.
- 데스크톱에서 닫기 액션을 누르면 상세를 닫고 목록 탐색 상태로 돌아간다.
- 모바일 상세는 modal이나 bottom sheet가 아니라 전체 화면 페이지로 표시한다.
- 모바일에서 화면의 뒤로 가기 버튼과 브라우저 기본 뒤로 가기는 모두 지도 화면으로 이동한다.
- 모바일에서 상세를 닫고 지도 화면으로 돌아갈 때 선택한 place를 유지한다.
- 모바일에서 상세를 닫고 지도 화면으로 돌아갈 때 지도는 선택한 place 위치를 기준으로 보여준다.
- 상세 상태는 `place_detail_load = idle | loading | ready | error`로 관리한다.
- 상세 로딩 중에는 필드 대신 진행 중 상태를 표시한다.
- 상세 로딩 실패 시 현재 상세 컨테이너에서 에러와 재시도 액션을 표시한다.
- 이번 1차 상세 UI에서는 네이버 지도 이동, 추천, 내 리뷰, 리뷰 작성 UI를 표시하지 않는다.

## Acceptance Criteria
- 상세 필드가 모두 표시된다.
- 데스크톱 상세가 목록 영역 내부 전환 방식으로 표시된다.
- 데스크톱 닫기 액션으로 목록 탐색 상태로 돌아간다.
- 모바일 상세는 전체 화면 페이지로 표시된다.
- 모바일 뒤로 가기 후 지도 화면에서 선택한 place가 유지된다.
- 모바일 뒤로 가기 후 지도는 선택한 place 위치를 기준으로 보여준다.
- 상세 로딩 중에는 진행 중 상태가 보인다.
- 상세 로딩 실패 시 재시도 액션을 확인할 수 있다.
- 네이버 이동 / 추천 / 내 리뷰 / 리뷰 작성 UI는 보이지 않는다.

## TDD Implementation Order
1. 상세 필드 표시 테스트를 작성한다.
2. 데스크톱 sidebar 내부 상세 전환 테스트를 작성한다.
3. 데스크톱 닫기 액션 테스트를 작성한다.
4. 모바일 전체 화면 상세 테스트를 작성한다.
5. 모바일 뒤로 가기 후 지도 복귀 테스트를 작성한다.
6. 선택 place 유지 테스트를 작성한다.
7. 상세 로딩 상태 테스트를 작성한다.
8. 상세 로딩 실패 상태 테스트를 작성한다.
9. 제외된 UI 미노출 테스트를 작성한다.
10. 구현한다.
11. 전체 테스트를 통과시킨다.
