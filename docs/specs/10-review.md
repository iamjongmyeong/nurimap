# Spec FR-10: Review

## Summary
리뷰 작성과 별점 평가 규칙, 상세 화면 표시 규칙을 정의한다.

## Scope
- 리뷰 작성
- 리뷰 작성 단계의 별점 평가
- 작성자/작성일 표시

## Functional Requirements
- 로그인 사용자만 리뷰를 작성할 수 있다.
- 리뷰 작성 단계에서 별점 평가를 함께 입력한다.
- 별점 입력 UI는 5개의 별 모양 버튼을 사용한다.
- 별점은 1점에서 5점 사이 정수만 허용한다.
- 이번 릴리즈에서는 리뷰와 함께 입력한 별점 평가를 수정할 수 없다.
- 리뷰는 place 상세에 작성자와 작성일과 함께 표시한다.
- 리뷰 수정/삭제는 이번 범위에 포함하지 않는다.

## Acceptance Criteria
- 로그인 사용자는 리뷰와 별점 평가를 함께 작성할 수 있다.
- 리뷰 작성 시 입력된 별점은 평균 별점과 별점 수에 반영된다.
- 리뷰는 작성자와 작성일과 함께 상세에 표시된다.

## TDD Implementation Order
1. 로그인 사용자 리뷰 작성 테스트를 작성한다.
2. 리뷰 작성 단계 별점 입력 테스트를 작성한다.
3. 별점 범위 검증 테스트를 작성한다.
4. 비로그인 차단 테스트를 작성한다.
5. 평균 별점 반영 테스트를 작성한다.
6. 작성자/작성일 표시 테스트를 작성한다.
7. 구현한다.
8. 전체 테스트를 통과시킨다.

## Required Test Cases
- 로그인 사용자 리뷰+별점 작성 성공
- 별점 1~5 외 값 실패
- 비로그인 사용자 실패
- 리뷰 저장 후 평균 별점/별점 수 갱신
- 작성자/작성일 표시

## Manual QA Checklist
- 리뷰 작성 시 별점 버튼을 함께 입력할 수 있다.
- 리뷰 작성 후 상세에 즉시 보인다.
- 평균 별점과 별점 수가 갱신된다.
- 작성자와 작성일이 함께 보인다.

## QA Evidence
- 테스트 실행 결과
- 리뷰/별점 수동 검증 결과
