# Spec FR-10: Review

## Summary
리뷰 작성 규칙과 상세 화면 표시 규칙을 정의한다.

## Scope
- 리뷰 작성
- 작성자/작성일 표시

## Functional Requirements
- 로그인 사용자만 리뷰를 작성할 수 있다.
- 리뷰는 place 상세에 작성자와 작성일과 함께 표시한다.
- 리뷰 수정/삭제는 이번 범위에 포함하지 않는다.

## Acceptance Criteria
- 로그인 사용자는 리뷰를 작성할 수 있다.
- 리뷰는 작성자와 작성일과 함께 상세에 표시된다.

## TDD Implementation Order
1. 로그인 사용자 리뷰 작성 테스트를 작성한다.
2. 비로그인 차단 테스트를 작성한다.
3. 작성자/작성일 표시 테스트를 작성한다.
4. 구현한다.
5. 전체 테스트를 통과시킨다.

## Required Test Cases
- 로그인 사용자 리뷰 작성 성공
- 비로그인 사용자 실패
- 작성자/작성일 표시

## Manual QA Checklist
- 리뷰 작성 후 상세에 즉시 보인다.
- 작성자와 작성일이 함께 보인다.

## QA Evidence
- 테스트 실행 결과
- 리뷰 수동 검증 결과
