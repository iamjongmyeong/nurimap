# Spec FR-08: Place Detail

## Summary
place 상세 화면에 표시해야 할 정보와 상세 화면 동작을 정의한다.

## Scope
- 상세 표시 필드
- 내 별점 상태 표시
- 리뷰 작성 UI 노출 조건
- 네이버 지도 이동 버튼
- 데스크톱 floating detail panel
- 모바일 상세 표현 방식
- 모바일 뒤로 가기 동작
- 상세 로딩/오류 상태

## Functional Requirements
- 상세 화면에는 이름, 주소, 종류, 평균 별점, 별점 수, 등록자, 추천 수, 리뷰를 표시한다.
- 화면에 표시하는 `별점 수`는 내부 집계 필드 `review_count`를 의미한다.
- 로그인 사용자의 내 별점 상태를 보여준다.
- `내 별점 상태`는 현재 사용자의 단일 review `rating_score`를 기준으로 표시한다.
- 현재 사용자가 아직 같은 place에 review를 작성하지 않았을 때만 리뷰 작성 UI를 표시한다.
- 현재 사용자가 이미 같은 place에 review를 작성한 경우 리뷰 작성 UI를 표시하지 않고 기존 내 리뷰와 내 별점 상태를 표시한다.
- 네이버 지도 이동 버튼을 제공한다.
- 데스크톱과 모바일에서 동일한 정보 구조를 유지한다.
- 데스크톱 상세는 지도 위에 떠 있는 floating panel로 표시한다.
- 데스크톱 상세는 너비 `390px`, 상단 inset `24px`, 하단 inset `24px`, 사이드바와 간격 `24px`를 사용한다.
- 데스크톱 상세의 높이 `calc(100vh - 48px)`는 상단 bar가 아니라 상단/하단 inset 합을 의미한다.
- 데스크톱 상세는 둥근 모서리와 그림자를 가진 카드처럼 보이고, 지도는 패널 뒤에서 계속 보여야 한다.
- 모바일 상세는 modal이나 bottom sheet가 아니라 전체 화면 페이지로 표시한다.
- 모바일에서 화면의 뒤로 가기 버튼과 브라우저 기본 뒤로 가기는 모두 지도 화면으로 이동한다.
- 모바일에서 상세를 닫고 지도 화면으로 돌아갈 때 선택한 place를 유지한다.
- 모바일에서 상세를 닫고 지도 화면으로 돌아갈 때 지도는 선택한 place 위치를 기준으로 보여준다.
- 상세 상태는 `place_detail_load = idle | loading | ready | error`로 관리한다.
- 상세 로딩 중에는 필드 대신 진행 중 상태를 표시한다.
- 상세 로딩 실패 시 현재 상세 컨테이너에서 에러와 재시도 액션을 표시한다.

## Acceptance Criteria
- 상세 필드가 모두 표시된다.
- 내 별점 상태가 보인다.
- 이미 review를 작성한 사용자는 리뷰 작성 UI를 보지 않는다.
- review가 없는 사용자는 리뷰 작성 UI를 볼 수 있다.
- 네이버 지도 이동 버튼이 동작한다.
- 데스크톱 상세가 지도 위에 떠 있는 floating panel로 표시된다.
- 데스크톱 상세 패널 뒤에 지도가 계속 보인다.
- 모바일 상세는 전체 화면 페이지로 표시된다.
- 모바일 뒤로 가기 후 지도 화면에서 선택한 place가 유지된다.
- 모바일 뒤로 가기 후 지도는 선택한 place 위치를 기준으로 보여준다.
- 상세 로딩 중에는 진행 중 상태가 보인다.
- 상세 로딩 실패 시 재시도 액션을 확인할 수 있다.

## TDD Implementation Order
1. 상세 필드 표시 테스트를 작성한다.
2. 내 별점 상태 테스트를 작성한다.
3. 리뷰 작성 UI 노출 조건 테스트를 작성한다.
4. 데스크톱 floating detail panel 레이아웃 테스트를 작성한다.
5. 네이버 지도 이동 버튼 테스트를 작성한다.
6. 모바일 전체 화면 상세 테스트를 작성한다.
7. 모바일 뒤로 가기 후 지도 복귀 테스트를 작성한다.
8. 선택 place 유지 테스트를 작성한다.
9. 상세 로딩 상태 테스트를 작성한다.
10. 상세 로딩 실패 상태 테스트를 작성한다.
11. 구현한다.
12. 전체 테스트를 통과시킨다.

- 등록자/추천 수/리뷰 표시
- 내 별점 상태 표시
- review 존재 여부에 따른 리뷰 작성 UI 노출 조건
- 데스크톱 floating detail panel 표시
- 데스크톱 상세 패널 뒤 지도 노출
- 네이버 지도 이동 버튼 링크
- 모바일 전체 화면 상세 표시
- 모바일 뒤로 가기 시 지도 화면 복귀
- 모바일 뒤로 가기 시 선택 place 유지
- 상세 로딩 상태 표시
- 상세 로딩 실패 시 재시도

## Manual QA Checklist
- 상세에서 필요한 정보가 모두 보인다.
- review가 있는 사용자는 리뷰 작성 UI가 보이지 않는다.
- review가 없는 사용자는 리뷰 작성 UI를 볼 수 있다.
- 데스크톱 상세가 지도 위에 떠 있는 카드처럼 보인다.
- 데스크톱 상세 패널 뒤에 지도가 계속 보인다.
- 네이버 이동 버튼이 올바른 링크를 연다.
- 모바일 상세가 전체 화면으로 열린다.
- 모바일 뒤로 가기 시 지도 화면으로 돌아간다.
- 모바일 뒤로 가기 후 선택한 place가 유지된다.
- 상세 로딩 중 진행 상태가 보인다.
- 상세 로딩 실패 시 재시도 액션이 보인다.
