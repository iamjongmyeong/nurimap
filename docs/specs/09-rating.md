# Spec FR-09: Rating

## Summary
사용자별 별점 추가 및 수정 규칙을 정의한다.

## Scope
- 별점 입력
- 사용자별 단일 별점
- 평균 별점/별점 수 집계

## Functional Requirements
- 별점은 1점에서 5점 사이 정수다.
- 입력 UI는 5개의 별 모양 버튼이다.
- 한 사용자는 place마다 하나의 rating만 가진다.
- 같은 사용자가 다시 평가하면 기존 rating을 수정한다.
- 평균 별점과 별점 수를 계산해 표시한다.

## Acceptance Criteria
- 로그인 사용자는 1~5점 별점을 남기거나 수정할 수 있다.
- 평균 별점과 별점 수가 정확히 갱신된다.

## TDD Implementation Order
1. 별점 범위 검증 테스트를 작성한다.
2. 사용자별 단일 별점 테스트를 작성한다.
3. 별점 수정 테스트를 작성한다.
4. 평균 별점 집계 테스트를 작성한다.
5. 구현한다.
6. 전체 테스트를 통과시킨다.

## Required Test Cases
- 1~5 외 값 실패
- 두 번째 별점은 수정 처리
- 평균 별점/별점 수 계산

## Manual QA Checklist
- 별 버튼으로 점수를 남길 수 있다.
- 기존 별점을 바꾸면 집계가 다시 계산된다.

## QA Evidence
- 테스트 실행 결과
- 별점 수동 검증 결과
