# Spec FR-04: Place Registration

## Summary
place 추가 입력 흐름과 저장 전 검증 규칙을 정의한다.

## Scope
- place 추가 UI
- 입력값 검증
- 초기 별점 기본값
- 초기 리뷰 생성 규칙
- 저장 실패 처리
- 저장 성공 후 이동
- 저장 비동기 상태

## Functional Requirements
- 입력값은 `place_type`, `zeropay_status`를 포함한다.
- 현재 사용자가 대상 place에 review가 없을 때만 초기 별점과 리뷰를 포함한다.
- `place_type = restaurant | cafe`
- `zeropay_status = available | unavailable | needs_verification`
- 초기 별점 UI는 별 모양 버튼을 사용한다.
- 초기 별점 기본값은 5점이다.
- 초기 별점은 1점에서 5점 사이 정수만 허용한다.
- 리뷰는 500자 이하로 제한한다.
- 최종 좌표를 확보한 경우에만 등록 흐름을 진행한다.
- place 등록 시 입력한 초기 별점과 리뷰는 등록자의 첫 Review 엔터티로 저장한다.
- 현재 사용자가 대상 place에 이미 review를 작성한 경우 초기 별점과 리뷰 입력 UI를 노출하지 않는다.
- 저장 중에는 제출 버튼을 비활성화한다.
- 저장 실패 시 place 추가 UI에 머물며 `place_type`, `zeropay_status`, 초기 별점, 리뷰 입력값을 유지한다.
- 저장 실패는 place 추가 UI 안에서 다시 시도 가능한 방식으로 표시한다.
- 등록 성공 시 별도 완료 화면은 두지 않고 결과 place 상세 화면으로 이동한다.
- 저장 상태는 `place_submit = idle | submitting | error`로 관리한다.

## Acceptance Criteria
- 초기 별점은 기본적으로 5점 선택 상태다.
- 잘못된 enum 값은 저장되지 않는다.
- 리뷰 500자 초과는 실패한다.
- 좌표 없는 place는 저장 단계로 넘어가지 않는다.
- 현재 사용자가 대상 place에 review가 없을 때만 초기 별점과 리뷰를 입력할 수 있다.
- place 등록 성공 시 입력한 초기 별점과 리뷰가 등록자의 첫 review로 저장된다.
- 현재 사용자가 대상 place에 이미 review가 있으면 초기 별점과 리뷰 입력 UI가 보이지 않는다.
- 저장 중에는 제출 버튼을 다시 누를 수 없다.
- 저장 실패 시 입력한 `place_type`, `zeropay_status`, 별점, 리뷰가 유지된다.
- 등록 성공 시 place 추가 UI가 닫히고 결과 place 상세 화면이 열린다.
- 저장 중에는 진행 중 상태가 보인다.

## TDD Implementation Order
1. enum 검증 테스트를 작성한다.
2. 초기 별점 기본값 테스트를 작성한다.
3. 별점 범위 테스트를 작성한다.
4. 리뷰 길이 제한 테스트를 작성한다.
5. 좌표 없는 등록 차단 테스트를 작성한다.
6. 초기 리뷰 생성 테스트를 작성한다.
7. 기존 review 존재 시 초기 리뷰 입력 UI 미노출 테스트를 작성한다.
8. 저장 중 진행 상태 테스트를 작성한다.
9. 저장 중 제출 버튼 비활성화 테스트를 작성한다.
10. 저장 실패 시 입력값 유지 테스트를 작성한다.
11. 등록 성공 후 상세 이동 테스트를 작성한다.
12. 구현한다.
13. 전체 테스트를 통과시킨다.

## Required Test Cases
- 초기 별점 기본값 5점
- 별점 1~5 외 값 실패
- 리뷰 500자 초과 실패
- 좌표 없는 place 등록 차단
- 초기 리뷰가 등록자의 첫 review로 저장됨
- 기존 review 존재 시 초기 리뷰 입력 UI 미노출
- 저장 중 제출 버튼 비활성화
- 저장 실패 시 입력값 유지
- 저장 중 진행 상태 표시
- 등록 성공 시 결과 place 상세 이동

- place 추가 화면에서 5번째 별이 기본 선택이다.

- review가 없는 경우에만 초기 리뷰 입력 UI가 보인다.
- place 등록 후 초기 리뷰와 별점이 첫 review로 반영된다.
- 저장 중 제출 버튼이 비활성화된다.
- 저장 중 진행 상태가 보인다.
- 저장 실패 시 입력한 값이 유지된다.
- 등록 성공 시 별도 완료 화면 없이 상세 화면으로 이동한다.
