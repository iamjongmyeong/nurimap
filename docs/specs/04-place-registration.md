# Spec FR-04: Place Registration

## Summary
place 추가 입력 흐름과 저장 전 검증 규칙을 정의한다.

## Scope
- place 추가 UI
- 입력값 검증
- 초기 별점 기본값
- 초기 리뷰 생성 규칙
- 기존 review 사용자 처리
- 저장 실패 처리
- 저장 성공 후 이동
- 저장 비동기 상태

## Functional Requirements
- place 추가 UI는 한 화면 안의 2단계 progressive disclosure로 구성한다.
- 1단계는 Naver Map URL 입력과 조회다.
- 2단계는 조회된 place 요약 확인과 등록 입력이다.
- 입력값은 `place_type`, `zeropay_status`를 포함한다.
- 2단계는 조회와 좌표 확보가 성공한 뒤에만 열린다.
- 2단계 상단에는 조회된 place `name`과 대표 주소를 읽기 전용 summary UI로 표시한다.
- 대표 주소는 `road_address`를 우선 사용하고, 없으면 `land_lot_address`를 사용한다.
- 조회된 place `name`과 대표 주소는 사용자가 수정할 수 없다.
- 현재 사용자가 대상 place에 review가 없을 때만 초기 별점과 리뷰를 포함한다.
- `place_type = restaurant | cafe`
- `zeropay_status = available | unavailable | needs_verification`
- 초기 별점 UI는 별 모양 버튼을 사용한다.
- 초기 별점 기본값은 5점이다.
- 초기 별점은 1점에서 5점 사이 정수만 허용한다.
- 리뷰는 500자 이하로 제한한다.
- 최종 좌표를 확보한 경우에만 등록 흐름을 진행한다.
- place 등록 시 입력한 초기 별점과 리뷰는 등록자의 첫 Review 엔터티로 저장한다.
- 현재 사용자가 대상 place에 이미 review를 작성한 경우 제목이 `이미 리뷰를 남긴 장소예요`인 modal을 표시한다.
- 이 modal은 본문 없이 제목만 사용한다.
- 이 modal의 CTA는 `장소 상세 보기`, `닫기`를 사용한다.
- `장소 상세 보기`를 누르면 place 등록 화면을 닫고 기존 place 상세 화면으로 이동한다.
- `닫기`를 누르면 place 등록 화면을 닫고 지도 탐색 상태로 돌아간다.
- 저장 중에는 제출 버튼을 비활성화한다.
- 저장 실패 시 place 추가 UI에 머물며 `place_type`, `zeropay_status`, 초기 별점, 리뷰 입력값을 유지한다.
- 저장 실패는 place 추가 UI 안에서 다시 시도 가능한 방식으로 표시한다.
- 등록 성공 시 place 추가 UI를 닫고 지도와 목록을 최신 상태로 갱신한 뒤 결과 place 상세 화면으로 이동한다.
- 등록 성공 시 별도 완료 화면은 두지 않고 결과 place 상세 화면으로 이동한다.
- 저장 상태는 `place_submit = idle | submitting | error`로 관리한다.

## Acceptance Criteria
- place 추가 UI는 별도 화면 전환 없이 같은 화면 안의 2단계로 진행된다.
- URL 조회와 좌표 확보가 성공해야만 등록 입력 단계가 열린다.
- 등록 입력 단계 상단에는 조회된 place 이름과 대표 주소가 읽기 전용으로 보인다.
- 초기 별점은 기본적으로 5점 선택 상태다.
- 잘못된 enum 값은 저장되지 않는다.
- 리뷰 500자 초과는 실패한다.
- 좌표 없는 place는 저장 단계로 넘어가지 않는다.
- 현재 사용자가 대상 place에 review가 없을 때만 초기 별점과 리뷰를 입력할 수 있다.
- place 등록 성공 시 입력한 초기 별점과 리뷰가 등록자의 첫 review로 저장된다.
- 현재 사용자가 대상 place에 이미 review가 있으면 제목이 `이미 리뷰를 남긴 장소예요`인 modal이 보인다.
- `장소 상세 보기`를 누르면 기존 place 상세 화면으로 이동한다.
- `닫기`를 누르면 place 등록 화면이 닫히고 지도 탐색 상태로 돌아간다.
- 저장 중에는 제출 버튼을 다시 누를 수 없다.
- 저장 실패 시 입력한 `place_type`, `zeropay_status`, 별점, 리뷰가 유지된다.
- 등록 성공 시 지도와 목록이 최신 상태로 갱신된다.
- 등록 성공 시 place 추가 UI가 닫히고 결과 place 상세 화면이 열린다.
- 저장 중에는 진행 중 상태가 보인다.

## TDD Implementation Order
1. 같은 화면 2단계 전환 테스트를 작성한다.
2. 읽기 전용 place 요약 표시 테스트를 작성한다.
3. enum 검증 테스트를 작성한다.
4. 초기 별점 기본값 테스트를 작성한다.
5. 별점 범위 테스트를 작성한다.
6. 리뷰 길이 제한 테스트를 작성한다.
7. 좌표 없는 등록 차단 테스트를 작성한다.
8. 초기 리뷰 생성 테스트를 작성한다.
9. 기존 review 존재 시 중복 리뷰 modal 표시 테스트를 작성한다.
10. 중복 리뷰 modal의 `장소 상세 보기` 이동 테스트를 작성한다.
11. 중복 리뷰 modal의 `닫기` 종료 테스트를 작성한다.
12. 저장 중 진행 상태 테스트를 작성한다.
13. 저장 중 제출 버튼 비활성화 테스트를 작성한다.
14. 저장 실패 시 입력값 유지 테스트를 작성한다.
15. 등록 성공 후 지도/목록 갱신 테스트를 작성한다.
16. 등록 성공 후 상세 이동 테스트를 작성한다.
17. 구현한다.
18. 전체 테스트를 통과시킨다.

- review가 없는 경우에만 초기 리뷰 입력 UI가 보인다.

- 같은 place에 이미 review가 있으면 `이미 리뷰를 남긴 장소예요` modal이 보인다.
- 중복 리뷰 modal의 `장소 상세 보기`로 기존 상세 화면으로 이동한다.
- 저장 중 제출 버튼이 비활성화된다.
- 저장 중 진행 상태가 보인다.
- 저장 실패 시 입력한 값이 유지된다.
- 등록 성공 후 지도와 목록에 새 place가 반영된다.
- 등록 성공 시 별도 완료 화면 없이 상세 화면으로 이동한다.
