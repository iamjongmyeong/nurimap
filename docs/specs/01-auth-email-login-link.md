# Spec FR-01: Auth Email Login Link

## Summary
회사 이메일 로그인 링크 인증을 정의한다. 이 문서는 인증 구현의 source of truth다.

## Scope
- 허용 도메인 검증
- 로그인 링크 발급
- 로그인 링크 검증
- 최초 로그인 후 이름 수집
- 세션 생성 및 복원
- 로그인 링크 요청 제한

## Functional Requirements
- 허용 도메인은 `@nurimedia.co.kr` 하나다.
- 허용 도메인이 아니면 로그인 링크를 발급하지 않는다.
- 로그인 이메일에는 로그인 전용 URL을 포함한다.
- 로그인 전용 URL은 직접 로그인 링크다.
- 로그인 전용 URL 유효시간은 5분이다.
- 새 로그인 링크를 발급하면 이전 링크는 무효화된다.
- 로그인 전용 URL은 한 번 사용하면 다시 사용할 수 없다.
- 동일 이메일 재요청 cooldown은 5분이다.
- 동일 이메일 로그인 링크 요청은 하루 최대 5회다.
- 로그인 성공 시 같은 브라우저에서 최대 90일 세션을 유지한다.
- 비로그인 사용자는 보호된 화면과 API에 접근할 수 없다.
- 로그인 성공 후 사용자 이름이 비어 있으면 이름 입력 화면으로 보낸다.
- 이름 입력은 단일 input field 하나로 받는다.
- 이름 input은 비울 수 없다.
- 이름은 1글자 이상 입력되면 유효하다.
- 이름 입력 완료 전에는 앱 메인 화면으로 진입할 수 없다.
- 이름이 이미 있는 사용자는 이름 입력 화면을 건너뛴다.

## Acceptance Criteria
- 허용 도메인이 아닌 이메일은 로그인 링크를 발급할 수 없다.
- 로그인 이메일에는 로그인 전용 URL이 포함된다.
- 로그인 전용 URL만으로 로그인이 가능하다.
- 로그인 전용 URL은 5분 뒤 만료된다.
- 새 로그인 링크를 발급하면 이전 링크는 실패한다.
- 같은 이메일은 5분 내 재요청할 수 없다.
- 같은 이메일은 하루 5회까지만 링크 요청이 가능하다.
- 사용한 링크는 다시 사용할 수 없다.
- 로그인 성공 시 세션이 생성되고 재접속 시 복구된다.
- 로그인 성공 후 이름이 비어 있으면 이름 입력 화면으로 이동한다.
- 이름 input은 한 개이며, 빈 값은 허용되지 않는다.
- 이름은 1글자 이상이면 저장할 수 있다.
- 이름 입력을 완료해야 앱 메인 화면으로 진입할 수 있다.
- 이름이 이미 저장된 사용자는 이름 입력 없이 앱으로 진입한다.

## TDD Implementation Order
1. 허용 도메인 검증 테스트를 작성한다.
2. 로그인 링크 발급 테스트를 작성한다.
3. 로그인 링크 만료 테스트를 작성한다.
4. 로그인 링크 재발급 무효화 테스트를 작성한다.
5. 재요청 제한 테스트를 작성한다.
6. 단일 사용 테스트를 작성한다.
7. 이름 없는 사용자 온보딩 리다이렉트 테스트를 작성한다.
8. 이름 입력 검증 테스트를 작성한다.
9. 이름 저장 후 앱 진입 테스트를 작성한다.
10. 세션 복원 테스트를 작성한다.
11. 보호 라우트 차단 테스트를 작성한다.
12. 구현한다.
13. 전체 테스트를 통과시킨다.

## Required Test Cases
- `@nurimedia.co.kr` 이메일은 허용된다.
- 다른 도메인은 거부된다.
- 로그인 링크는 발급 후 5분까지만 유효하다.
- 새 링크 발급 시 이전 링크는 실패한다.
- 같은 이메일은 5분 내 재요청할 수 없다.
- 같은 이메일은 하루 5회를 초과할 수 없다.
- 사용한 링크는 재사용할 수 없다.
- 비로그인 사용자는 보호 화면/API에 접근할 수 없다.
- 이름이 없는 사용자는 로그인 후 이름 입력 화면으로 이동한다.
- 이름이 있는 사용자는 로그인 후 바로 앱으로 진입한다.
- 이름 input이 빈 값이면 저장할 수 없다.
- 이름 input에 1글자 이상 입력하면 저장할 수 있다.

## Manual QA Checklist
- 허용 도메인 이메일로 로그인 링크 요청이 된다.
- 허용 도메인이 아닌 이메일로 요청이 막힌다.
- 이메일에서 로그인 링크를 클릭하면 로그인된다.
- 만료된 링크는 실패한다.
- 재발급 후 이전 링크는 실패한다.
- 이름이 없는 계정은 로그인 후 이름 입력 화면으로 이동한다.
- 이름 input 하나만 보이고, 빈 값으로는 제출할 수 없다.
- 1글자 이상 입력하면 저장되고 앱으로 진입한다.
- 로그아웃 후 보호 화면 접근이 막힌다.
