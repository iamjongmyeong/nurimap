# User Flow

This document is the source of truth for user flows.

관련 문서: [Design Foundations](../04-design/foundations.md)

## Authentication Flow
1. 사용자가 웹사이트에 접속한다.
2. 로그인 화면에서 회사 이메일을 입력한다.
3. 시스템이 허용 도메인인지 확인한다.
4. 허용 도메인이 아니거나 로그인 링크를 요청할 수 없는 상태면 로그인 화면에 그대로 머물며 실패 이유를 확인한다.
5. 요청이 가능하면 이메일로 로그인 전용 URL을 발송한다.
6. 사용자가 로그인 전용 URL을 클릭한다.
7. 로그인 전용 URL이 유효하면 사용자 이름 존재 여부를 확인한다.
8. 로그인 전용 URL이 만료되었거나, 이미 사용되었거나, 새 링크 발급으로 무효화되었으면 인증 실패 화면을 본다.
9. 인증 실패 화면에서 `새 로그인 링크 받기` 또는 `이메일 다시 입력`을 선택한다.
10. 이름이 비어 있으면 이름 입력 화면으로 이동한다.
11. 이름 입력이 완료되면 장소 탐색 화면으로 이동한다.
12. 같은 브라우저 재방문 시 저장 세션이 유효하면 로그인 화면을 건너뛴다.

로그인 링크 규칙:
- 로그인 전용 URL은 직접 로그인에 사용할 수 있다.
- 로그인 전용 URL은 5분 동안만 유효하다.
- 새 로그인 링크를 발급하면 이전 로그인 전용 URL은 무효화된다.
- 로그인 전용 URL은 한 번 사용하면 다시 사용할 수 없다.

인증 실패 처리 규칙:
- `비허용 도메인`, `재요청 cooldown`, `일일 요청 횟수 초과`는 로그인 화면에서 처리한다.
- 로그인 화면에서 인증 요청이 실패해도 사용자가 입력한 이메일은 유지한다.
- `만료된 링크`, `이미 사용된 링크`, `무효화된 링크`는 인증 실패 화면에서 처리한다.
- 인증 실패 화면에서는 앱 메인 화면으로 진입할 수 없다.
- 인증 실패 화면의 기본 CTA는 `새 로그인 링크 받기`다.
- 인증 실패 화면의 보조 CTA는 `이메일 다시 입력`이다.

## Name Capture Flow
1. 로그인 성공 후 이름이 비어 있는 사용자는 이름 입력 화면을 본다.
2. 이름 입력은 성과 이름을 포함한 단일 input field 하나로 받는다.
3. 이름 input은 비울 수 없다.
4. 1글자 이상 입력하면 제출할 수 있다.
5. 저장 성공 후 장소 탐색 화면으로 이동한다.

## Place Submission Flow
1. 로그인 사용자가 장소 추가 UI를 연다.
2. 같은 장소 추가 UI 안의 1단계에서 사용자가 Naver Map URL을 입력한다.
3. 입력값이 Naver Map URL이 아니면 inline error로 `네이버 지도 URL을 입력해주세요.` 문구를 보고 같은 화면에 머문다.
4. 시스템이 URL을 정규화하고 장소 데이터를 조회한다.
5. 조회에 실패하면 실패 안내를 보고 `다시 시도` 또는 `닫기`를 선택한다.
6. 시스템은 좌표 확보를 아래 순서로 시도한다.
   - 네이버 응답의 위도/경도
   - `road_address` geocoding
   - `land_lot_address` geocoding
7. 좌표 확보에 실패하면 실패 안내를 보고 `다시 시도` 또는 `닫기`를 선택한다.
8. 조회와 좌표 확보에 성공하면 시스템이 중복 장소와 현재 사용자의 기존 review 여부를 확인한다.
9. 현재 사용자가 이미 같은 장소에 review를 작성한 경우 중복 리뷰 안내를 본다.
10. 사용자가 `장소 상세 보기`를 누르면 장소 등록 화면을 닫고 기존 장소 상세 화면으로 이동한다.
11. 사용자가 `닫기`를 누르면 장소 등록 화면을 닫고 지도 탐색 상태로 돌아간다.
12. 현재 사용자의 review가 없으면 같은 장소 추가 UI 안의 2단계가 열린다.
13. 2단계 상단에서 조회된 장소명과 주소를 읽기 전용으로 확인한다.
14. 사용자는 `place_type`, `zeropay_status`, 초기 별점, 리뷰를 입력한다.
15. 저장 중에는 제출 버튼이 비활성화되고 진행 중 상태를 본다.
16. 저장에 실패하면 장소 추가 UI에 머물며 입력값을 유지한 채 실패 이유를 확인하고 다시 시도한다.
17. 신규 장소 생성에 성공하면 장소 추가 UI를 닫고 지도와 목록을 갱신한 뒤 새 장소 상세 화면을 연다.
18. 기존 장소 병합에 성공하면 장소 추가 UI를 닫고 지도와 목록을 갱신한 뒤 병합된 기존 장소 상세 화면을 연다.

장소 등록 결과 처리 규칙:
- 장소 추가 UI는 같은 화면 안의 2단계로 진행한다.
- 1단계는 Naver Map URL 입력과 조회다.
- 2단계는 조회된 장소 정보 확인과 등록 입력이다.
- URL 형식 오류는 URL 입력 필드 아래 inline error로 표시한다.
- 조회 실패와 좌표 확보 실패 시 사용자가 입력한 URL은 유지한다.
- 조회 성공 후 표시하는 장소명과 주소는 읽기 전용이다.
- 조회 성공 후 표시하는 대표 주소는 `road_address`를 우선 사용하고, 없으면 `land_lot_address`를 사용한다.
- 현재 사용자가 이미 같은 장소에 review를 작성한 경우 새 review를 추가하지 않고 기존 review를 유지한다.
- 초기 별점과 리뷰는 현재 사용자가 해당 장소에 review가 없을 때만 입력한다.
- 장소 등록 시 입력한 초기 별점과 리뷰는 등록자의 첫 review로 저장한다.
- 저장 실패 시 `place_type`, `zeropay_status`, 별점, 리뷰 입력값을 유지한다.
- 장소 등록 성공 시 별도 완료 화면은 두지 않는다.

## Browse Flow
1. 사용자가 지도 화면을 본다.
2. 데스크톱에서는 목록에서 장소를 선택하거나 지도 마커를 클릭하고, 모바일에서는 `목록 보기` 버튼으로 장소 목록 페이지로 이동하거나 지도 마커를 클릭한다.
3. 목록에서는 평균 별점, 리뷰 수, 제로페이 가능 여부를 함께 확인할 수 있다.
4. 데스크톱에서는 지도 위에 떠 있는 floating 상세 패널이 열리고, 모바일에서는 전체 화면 상세 페이지가 열린다.
5. 상세에서 평균 별점, 별점 수, 내 별점 상태, 등록자, 추천 수, 리뷰, 네이버 지도 이동 버튼을 본다.
6. 데스크톱에서는 닫기로, 모바일에서는 화면 상단 뒤로 가기 버튼 또는 브라우저 기본 뒤로 가기로 지도 화면으로 돌아간다.

모바일 장소 상세 화면 전환 규칙:
- 모바일 장소 상세 화면은 전체 화면 페이지로 연다.
- 모바일에서 지도 마커를 눌러 장소 상세 화면으로 이동할 수 있다.
- 모바일에서 장소 목록 페이지의 장소를 눌러 장소 상세 화면으로 이동할 수 있다.
- 모바일 장소 상세 화면에서 지도 화면으로 돌아갈 때 선택한 장소는 유지한다.
- 모바일 장소 상세 화면에서 지도 화면으로 돌아갈 때 지도는 선택한 장소 위치를 기준으로 보여준다.

## Recommendation Flow
1. 로그인 사용자가 상세 화면의 추천 버튼을 누른다.
2. 기존 추천이 없으면 추천을 추가한다.
3. 기존 추천이 있으면 추천을 취소한다.
4. 추천 수가 갱신된다.

## Review Flow
1. 로그인 사용자가 장소 상세 화면을 연다.
2. 현재 사용자가 아직 같은 장소에 review를 작성하지 않았으면 리뷰 작성 UI를 본다.
3. 리뷰 작성 단계에서 별점 평가를 함께 입력한다.
4. 시스템이 리뷰 내용과 별점이 유효한지 확인한다.
5. 저장 후 최신 리뷰 목록과 평균 별점, 별점 수, 내 별점 상태를 다시 보여준다.
6. 현재 사용자가 이미 같은 장소에 review를 작성한 경우 새 리뷰 작성 UI를 보지 않고 기존 내 별점 상태와 리뷰를 본다.
7. 작성자와 작성일이 함께 표시된다.
